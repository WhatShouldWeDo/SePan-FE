# Democrasee — UX 플로우 문서

> 최종 수정: 2026-02-08
> 상태: 초안

---

## 1. 전체 사용자 플로우 개요

```
[로그인/회원가입] → [대시보드 홈] → [지역분석] → [정책개발]
                         ↑                 ↓
                         └── 정책개발에서 지역분석 참조 가능
```

### 핵심 원칙

- 사용자는 항상 **대시보드 홈**을 거쳐 각 기능으로 이동
- 지역분석 → 정책개발은 **자연스러운 연결**(CTA 버튼)로 유도
- 모든 화면에서 **뒤로가기/홈으로** 돌아갈 수 있는 네비게이션 제공
- 데이터가 없는 상태(Empty State)도 항상 고려

## 2. 인증 플로우

### 2-1. 로그인

```
[로그인 페이지]
  ├── 아이디 입력
  ├── 비밀번호 입력
  ├── [로그인] 버튼 클릭
  │   ├── 성공 → [대시보드 홈]으로 이동
  │   └── 실패 → 에러 메시지 표시 (인라인)
  ├── [회원가입] 링크 → 회원가입 플로우로 이동
  └── [비밀번호 찾기] 링크 → (P1, MVP에서는 안내 텍스트만)
```

**UX 포인트:**
- 에러 메시지는 폼 상단 또는 해당 필드 아래에 한 번에 하나만 표시
- 로그인 버튼은 필수 필드가 모두 채워졌을 때만 활성화 (선택사항)
- 로딩 중에는 버튼에 스피너 표시, 중복 클릭 방지

### 2-2. 회원가입

```
[Step 1: 기본 정보]
  ├── 아이디, 비밀번호, 비밀번호 확인, 이름
  └── [다음] → Step 2로

[Step 2: 휴대폰 인증]
  ├── 휴대폰 번호 입력
  ├── [인증번호 발송] → SMS 발송
  ├── 인증번호 입력
  └── [인증 확인] → Step 3로

[Step 3: 승인코드 입력]
  ├── 관리자로부터 받은 승인코드 입력
  ├── [확인] → 유효성 검증
  │   ├── 유효 → Step 4로
  │   └── 무효 → 에러 메시지
  └── 승인코드가 없는 경우 안내 텍스트

[Step 4: 역할/지역/추가정보]
  ├── 역할 선택 (후보자, 사무장, 회계책임자, 사무원)
  ├── 지역 선택 (시·도 → 시·군·구 → 행정동/선거구)
  ├── 추가 정보 입력 (선택 항목)
  └── [가입 완료] → 환영 메시지 → [대시보드로 이동]
```

**UX 포인트:**
- 각 Step에 진행률 표시 (Step indicator: 1/4, 2/4...)
- 이전 Step으로 돌아가기 가능 (입력값 유지)
- 유효성 검사는 "다음" 버튼 클릭 시 또는 필드 blur 시 실행
- 마지막 Step 완료 시 간결한 환영 메시지 표시

## 3. 대시보드 홈 플로우

```
[대시보드 홈]
  ├── [상단] 오늘의 일정 인포배너
  │     └── 클릭 → (일정 상세, P2에서 구현)
  ├── [상단] 선거일 D-day 배지
  ├── [중단] 지역분석 요약 카드
  │     ├── 최다 민원 지역, 주요 민원 유형, 증가율
  │     └── [자세히 보기] → 지역분석 메인
  ├── [중단] 정책개발 요약 카드
  │     ├── 최신/진행중/완료된 정책 수
  │     └── [자세히 보기] → 정책개발 개요
  └── [하단] 퀵메뉴
        ├── 지역분석 바로가기 → 지역분석 메인
        ├── 정책개발 바로가기 → 정책개발 개요
        └── 일정 관리 → (P2)
```

**Empty State:**
- 캠프가 아직 지역을 설정하지 않은 경우: "지역을 설정하고 분석을 시작하세요" + 설정 CTA
- 정책이 아직 없는 경우: "첫 번째 정책을 만들어보세요" + 정책 생성 CTA

## 4. 지역분석 플로우

```
[지역분석 메인]
  ├── 지역 선택 (현재 설정된 지역 기본 표시)
  ├── 지표 카테고리 선택 (인구, 민원, 예산 등)
  └── [분석 시작] 또는 자동 로드

[지역분석 결과]
  ├── [현황 섹션]
  │     ├── 핵심 지표 카드 (현재값, 전년 대비 ↑↓, 전국 평균)
  │     ├── 연도별 추이 그래프 (라인 차트)
  │     └── 상세 데이터 표
  ├── [인접 지역 비교 섹션]
  │     ├── 내 지역 vs 인접 N개 지역 비교 테이블
  │     ├── 바 차트 (주요 지표별)
  │     └── 지역 클릭 → 해당 지역 상세로 이동
  ├── [AI 인사이트 섹션] (MVP: 목업)
  │     ├── 핵심 요약 문장 (1~3줄)
  │     └── [자세히 보기] → (P2에서 상세 AI 분석)
  └── [CTA 영역]
        ├── [상세 리포트 다운로드] → (P2, MVP에서는 비활성)
        └── [이 데이터로 정책 개발하기] → 정책개발로 이동
```

**UX 포인트:**
- 그래프와 표는 항상 **한 문장 요약**과 함께 제시 ("전년 대비 12% 증가")
- 로딩 중에는 차트/표 영역에 Skeleton UI 표시
- 인접 지역 비교는 기본 3~5개 지역, 더 보기로 확장

## 5. 정책개발 플로우

### 5-1. 탭 구조

```
[정책개발]
  ├── [개요 탭] (기본 진입점)
  ├── [역대공약분석 탭]
  ├── [확정공약 탭]
  └── [AI 추천 탭]
```

### 5-2. 개요 탭

```
[개요]
  ├── 카테고리별 공약 현황 (진행중 N개, 완료 N개 등)
  ├── 지역분석 요약 연결 (최신 분석 결과 미리보기)
  ├── 안내 텍스트: "지역분석 → 역대공약 참고 → AI 추천 → 확정" 플로우 설명
  └── CTA 버튼들
        ├── [지역분석 보기] → 지역분석
        ├── [AI 추천 받기] → AI 추천 탭
        └── [새 공약 작성] → 공약 작성 폼
```

### 5-3. 역대공약분석 탭

```
[역대공약분석]
  ├── 필터 영역
  │     ├── 선거 종류 (대선, 총선, 지선 등)
  │     ├── 정당 (드롭다운/멀티셀렉트)
  │     ├── 선거구
  │     └── 후보자명 검색
  ├── 결과 보기 모드
  │     ├── [리스트] → 공약 목록 (카드 형태)
  │     ├── [통계] → 카테고리별 분포, 키워드 빈도
  │     └── [정당별 비교] → 정당 간 공약 비교 표
  └── 공약 카드 클릭 → 공약 상세 모달/페이지
```

### 5-4. 확정공약 탭

```
[확정공약]
  ├── 확정된 공약 리스트 (카테고리별 그루핑)
  ├── 각 공약 카드
  │     ├── 제목, 카테고리, 상태, 요약
  │     ├── [수정] → 편집 폼
  │     └── [상세 보기] → 상세 페이지
  └── [새 공약 작성] 버튼 → 공약 작성 폼
```

### 5-5. AI 추천 탭 (MVP: 목업 수준)

```
[AI 추천]
  ├── AI 추천 공약 리스트
  │     ├── 추천 근거 (지역분석 데이터 기반)
  │     └── 각 추천 항목
  │           ├── 제목, 요약, 추천 이유
  │           ├── [상세 보기] → 상세 모달
  │           ├── [수정 후 채택] → 편집 폼 (채택 시 확정공약으로 이동)
  │           └── [건너뛰기] → 다음 추천
  └── 주의: 자동 저장/자동 채택 없음. 반드시 사용자 확인 필요.
```

## 6. 공통 네비게이션

```
[사이드바 / 상단 GNB]
  ├── 로고 (클릭 → 대시보드 홈)
  ├── 대시보드
  ├── 지역분석
  ├── 정책개발
  ├── (일정 관리 — P2)
  ├── (설정 — P2)
  └── 프로필/로그아웃
```

**UX 포인트:**
- 현재 위치는 네비게이션에서 시각적으로 구분 (active state)
- 모바일에서는 햄버거 메뉴로 축소
- 브레드크럼: 지역분석 > 결과 > 인접비교 (depth 2 이상일 때)
