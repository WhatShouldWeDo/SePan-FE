# Democrasee — 제품 요구사항 문서 (PRD)

> 최종 수정: 2026-02-08
> 상태: 초안 (MVP 정의 중)

---

## 1. 제품 비전

선거 캠프가 **지역 현황을 데이터로 파악**하고, 그에 기반한 **정책(공약)을 개발**할 수 있는 구독형 SaaS.

### 핵심 가치

- 복잡한 공공데이터를 **한눈에 이해할 수 있는 시각화**로 제공
- 데이터 기반 의사결정을 통해 **근거 있는 공약 개발** 지원
- 40~60대 장년층도 쉽게 사용할 수 있는 **직관적 UX**

---

## 2. 타겟 사용자

| 역할 | 설명 | 핵심 니즈 |
|------|------|-----------|
| 후보자 | 선거에 출마하는 본인 | 지역 현황 파악, 공약 확정 |
| 선거사무장 | 캠프 총괄 운영자 | 전체 상황 모니터링, 전략 수립 |
| 회계책임자 | 선거 자금 관리 | (추후 확장) |
| 선거사무원 | 실무 담당자 | 데이터 조회, 리포트 생성 |

### 사용자 특성

- 연령대: 40~60대 (디지털 리터러시가 상대적으로 낮을 수 있음)
- 환경: 데스크톱 우선, 태블릿/모바일에서도 확인 가능해야 함
- 사용 빈도: 선거 기간 중 매일, 비선거 기간에는 간헐적

## 4. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 접근성 | WCAG AA 이상 지향 (특히 색상 대비, 텍스트 크기) |

| 보안 | JWT 기반 인증, HTTPS 필수 |

## 5. MVP 범위 정의

### 포함 (In Scope)

- 로그인/회원가입 전체 플로우
- 대시보드 홈 (일정, D-day, 요약 카드, 퀵메뉴)
- 지역분석 기본 시각화 (현황, 추이, 인접 비교)
- 정책개발 기본 기능 (개요, 역대 공약, 확정 공약 CRUD)

### 미포함 (Out of Scope — 향후 확장)

- AI 기능 (인사이트, 추천) → 목업 UI만 구현
- 지도 폴리곤 실제 렌더링 → 추상화 인터페이스만
- 상세 리포트 생성/다운로드
- 알림/푸시
- 결제/구독 관리
- 관리자 대시보드

## 6. 용어 정의

| 용어 | 정의 |
|------|------|
| 캠프 | 선거 캠프. 하나의 후보자를 중심으로 구성된 팀/조직 |
| 선거구 | 선거가 실시되는 행정 구역 단위 |
| 행정동 | 지방자치단체의 행정 업무 처리를 위해 설정한 구역 |
| 확정공약 | 캠프에서 최종 채택한 공약 |
| AI 추천 | AI가 지역분석 데이터를 기반으로 제안하는 공약 후보 |
